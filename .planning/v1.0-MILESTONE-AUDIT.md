### Broken Flow

## Assessment

These integration gaps are **architectural seams** — each subsystem is complete and tested, but the glue code connecting them at runtime is missing. This is a natural consequence of building foundation-first (graph engine → agents → LLM → context → pipeline → API → UI) where each phase validates its own contracts.

**To make the system fully functional at runtime**, a follow-up integration phase would need to:
1. Wire LLMService into BaseAgent.act() dispatch
2. Wire ContextAdapter into AgentNode pre-execution
3. Connect API pipeline start to Temporal client
4. Replace execute_phase_activity TODO with real agent dispatch
5. Inject WorktreePool into AgentNode at graph construction
6. Fix dashboard event name mapping

These are estimated at 1-2 plans of integration work.
