       - `state_updates` key: `"tester_output"`

    **2. debugger.py** -- `@register_agent(AgentType.DEBUGGER)`, class `DebuggerAgent`:
       - `tools: ["file_read", "file_write", "file_edit", "bash", "test_runner", "stack_trace_analyzer", "root_cause_analyzer", "security_fix_generator"]`
       - `use_worktree: bool = True`
       - `perceive`: extracts `tester_output` (failed tests), `security_auditor_output` (security findings for DBUG-04), `qa_results`
       - `act`: returns `{"root_cause_analysis": ..., "fix_patches": [...], "security_fixes": [...], "retest_results": {"passed": int, "failed": int}, "iterations": int}`
       - `review`: validates `root_cause_analysis` exists and (`retest_results.failed == 0` OR `iterations >= max_retries`). `review_passed` is True when `retest_results.failed == 0`
       - `state_updates` key: `"debugger_output"`
       - SYSTEM_PROMPT: role = debugger and fixer, performs root cause analysis on test failures, generates fix proposals, applies patches, re-runs tests in fix-test loop until pass or max retries (3). DBUG-04: also parses SecurityAuditorAgent findings and generates security-specific fixes (input validation, SQL injection prevention, XSS mitigation, secret removal).
       - Include `max_fix_iterations: int = 3`
