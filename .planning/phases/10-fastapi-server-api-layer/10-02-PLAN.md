3. **Create `services/pipeline_service.py`** with `PipelineService` class:
   - `__init__(self, db: AsyncSession) -> None`
   - `VALID_TRANSITIONS` class variable: dict mapping `PipelineStatus` to a list of valid target states. Specifically: `PENDING -> [RUNNING, CANCELLED]`, `RUNNING -> [PAUSED, COMPLETED, FAILED, CANCELLED]`, `PAUSED -> [RUNNING, CANCELLED]`. No transitions from COMPLETED, FAILED, or CANCELLED.
   - `DEFAULT_PRESET_PHASES` class variable: dict mapping mode names to phase lists. This serves as the **fallback** when YAML files are not available.
     - `"full"`: `["brainstorming", "planning", "research", "architecture", "design", "implementation", "review", "testing", "debug_fix", "documentation"]`
     - `"quick"`: `["planning", "implementation", "testing"]`
     - `"review_only"`: `["review", "testing"]`
   - `@classmethod def load_preset_phases(cls) -> dict[str, list[str]]` -- Loads pipeline presets from `configs/pipelines/` YAML files. Tries to load `configs/pipelines/full.yaml`, `configs/pipelines/quick.yaml`, `configs/pipelines/review_only.yaml` using `yaml.safe_load()`. Each YAML file should have a `phases` key containing a list of phase name strings. If a YAML file exists and parses correctly, its phases override the corresponding `DEFAULT_PRESET_PHASES` entry. If YAML loading fails (file not found, parse error), fall back to `DEFAULT_PRESET_PHASES` for that preset and log a debug-level warning. Import `yaml` -- add `"pyyaml>=6.0"` to pyproject.toml if not already present (it is likely already present). Use `pathlib.Path` to resolve `configs/pipelines/` relative to the project root (two levels up from `services/pipeline_service.py`, or use a `CODEBOT_CONFIG_DIR` setting if available in config.py).
   - `PRESET_PHASES` class variable: set to `load_preset_phases()` at class definition time. This makes presets load once on import.
   - Map of phase name strings to `PhaseType` enum: `{"brainstorming": PhaseType.BRAINSTORMING, "planning": PhaseType.PLANNING, "research": PhaseType.RESEARCH, "architecture": PhaseType.ARCHITECTURE, "design": PhaseType.DESIGN, "implementation": PhaseType.IMPLEMENTATION, "review": PhaseType.REVIEW, "testing": PhaseType.TESTING, "debug_fix": PhaseType.DEBUG_FIX, "documentation": PhaseType.DOCUMENTATION, "deployment": PhaseType.DEPLOYMENT, "delivery": PhaseType.DELIVERY}`
   - `async create(self, project_id: UUID, payload: PipelineCreate) -> Pipeline` -- creates Pipeline record with `project_id`, `status=PipelineStatus.PENDING`. Stores `payload.mode` and `payload.config` in `graph_definition` as `{"mode": mode, "config": config}`. Determines phase list: if `payload.phases` is provided use it, else use `PRESET_PHASES[payload.mode]`. For each phase name in the list, creates a `PipelinePhase` record with `phase_type` from the name-to-enum map, `order` as the index, `status=PhaseStatus.PENDING`. Commits and returns pipeline.
   - `async get(self, pipeline_id: UUID) -> Pipeline | None` -- `await self._db.get(Pipeline, pipeline_id)`.
   - `async get_with_phases(self, pipeline_id: UUID) -> Pipeline | None` -- uses `selectinload(Pipeline.phases)` to eager-load phases.
   - `async list_for_project(self, project_id: UUID, *, page: int = 1, per_page: int = 20, status: str | None = None) -> tuple[list[Pipeline], int]` -- builds query with optional status filter (convert string to PipelineStatus), returns paginated results.
   - `async transition(self, pipeline: Pipeline, target_status: PipelineStatus) -> Pipeline` -- validates transition against `VALID_TRANSITIONS`. If invalid, raises `HTTPException(400, f"Cannot transition from {pipeline.status.value} to {target_status.value}")`. Updates `pipeline.status`. If `target_status == RUNNING` and `pipeline.started_at` is None, sets `started_at`. If `target_status in (COMPLETED, FAILED, CANCELLED)`, sets `completed_at`. Commits and returns.
