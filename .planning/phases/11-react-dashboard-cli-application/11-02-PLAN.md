<objective>
Build the CodeBot CLI application using Click + Rich + httpx that provides project creation, pipeline control, agent monitoring, and preset selection from the terminal.

Purpose: The CLI is the primary developer interface for headless operation and CI/CD integration. It provides terminal-based pipeline control without requiring the web dashboard. All four CLI requirements (CLI-01 through CLI-04) are implemented as a standalone Python package.

Output: A working `codebot` CLI with subcommands for project management (create, list, delete), pipeline control (start, pause, resume, stop, status), agent monitoring (list, logs streaming), and configuration (preset selection). Includes Rich-formatted output and a full test suite.
</objective>

<interfaces>
<!-- Phase 10 REST API endpoints the CLI consumes -->

Auth:
```
POST /api/v1/auth/login {email, password} -> {data: {access_token, refresh_token, token_type, expires_in, user}}
```

Projects:
```
POST /api/v1/projects {name, description, prd_content} -> {data: ProjectResponse} (201)
GET /api/v1/projects?page=1&per_page=20 -> {data: [ProjectResponse], pagination: {...}}
DELETE /api/v1/projects/{id} -> 204
```

Pipelines:
```
POST /api/v1/projects/{project_id}/pipelines {mode: "full"|"quick"|"review_only"} -> {data: PipelineResponse} (201)
POST /api/v1/pipelines/{id}/start -> {data: PipelineResponse}
POST /api/v1/pipelines/{id}/pause -> {data: PipelineResponse}
POST /api/v1/pipelines/{id}/resume -> {data: PipelineResponse}
POST /api/v1/pipelines/{id}/stop -> {data: PipelineResponse}
```

Agents:
```
GET /api/v1/agents?pipeline_id=... -> {data: [AgentResponse], pagination: {...}}
```

WebSocket:
```
/agents namespace: agent:status, agent:log, agent:metrics, agent:output
```

Response envelope:
```json
{"status": "success", "data": {...}, "meta": {"request_id": "...", "timestamp": "..."}}
{"status": "error", "error": {"code": "...", "message": "..."}}
```

**5. Create `apps/cli/codebot/client/api.py`** -- `CodeBotClient` class:
- Constructor: `__init__(self, base_url: str, token: str | None = None)` creates an `httpx.AsyncClient` with base_url, Content-Type header, and optional Bearer token
- Internal `_request(method, path, **kwargs)` method: makes HTTP request, returns parsed JSON dict. On status >= 400, raises `CodeBotAPIError(status_code, message)` extracted from response error envelope. On 204, returns `{"status": "success", "data": None}`.
- Methods: `login(email, password)`, `create_project(name, description, prd_content)`, `list_projects(page, per_page)`, `get_project(project_id)`, `delete_project(project_id)`, `create_pipeline(project_id, mode)`, `list_pipelines(project_id)`, `start_pipeline(pipeline_id)`, `pause_pipeline(pipeline_id)`, `resume_pipeline(pipeline_id)`, `stop_pipeline(pipeline_id)`, `list_agents(pipeline_id)`, `get_agent(agent_id)`
- `close()` method calls `self._client.aclose()`
- `CodeBotAPIError(Exception)` with `status_code: int` and `message: str` fields

**6. Create `apps/cli/codebot/client/streaming.py`** -- WebSocket log streaming:
- `async def stream_logs(base_url, pipeline_id, token, console)`: Connects via `websockets.connect` to `ws://{host}/ws/agents`. Sends subscribe message with pipeline_id. Loops over incoming messages, parsing JSON. For `agent:log` events, prints with Rich styling: timestamp (dim), level (color-coded: info=blue, warn=yellow, error=red, debug=dim), agent_id (bold), message. For `agent:status` events, prints status change. Catches `ConnectionRefusedError` and `KeyboardInterrupt` gracefully.

**1. Create `apps/cli/codebot/commands/__init__.py`** -- empty.

**2. Create `apps/cli/codebot/commands/project.py`** -- project management:
- `project_group = click.group("project")` with docstring "Manage CodeBot projects."
- `create` command: `--name` (prompt), `--description` (prompt, default ""), `--prd` (file path, optional). Reads PRD file if provided. Creates `CodeBotClient` from ctx.obj, calls `client.create_project(name, description, prd_content)` inside `asyncio.run()`. Wraps in `spinner("Creating project...")`. On success, prints project ID and name in green. Catches `CodeBotAPIError` and prints error in red. Always calls `client.close()`.
- `list_cmd` command (named "list"): `--page` (default 1), `--per-page` (default 20). Calls `client.list_projects()`, formats with `format_project_table`. Shows pagination info.
- `delete` command: takes `project_id` argument, `@click.confirmation_option`. Calls `client.delete_project()`. Prints confirmation.

**3. Create `apps/cli/codebot/commands/pipeline.py`** -- pipeline control:
- `pipeline_group = click.group("pipeline")` with docstring "Control pipeline execution."
- `start` command: `project_id` argument, `--preset` option (choices: full, quick, review-only, default full). Maps "review-only" to "review_only" for API. Calls `client.create_pipeline(project_id, mode)` then `client.start_pipeline(pipeline_id)`. Formats output with `format_pipeline_status`.
- `pause` command: `pipeline_id` argument. Calls `client.pause_pipeline()`. Prints status.
- `resume` command: `pipeline_id` argument. Calls `client.resume_pipeline()`. Prints status.
- `stop` command: `pipeline_id` argument, `--yes` confirmation. Calls `client.stop_pipeline()`.
- `status` command: `pipeline_id` argument. Fetches pipeline via API, formats with `format_pipeline_status`.
- `list_cmd` command (named "list"): `project_id` argument. Lists pipelines for project.

**4. Create `apps/cli/codebot/commands/agent.py`** -- agent monitoring:
- `agent_group = click.group("agent")` with docstring "Monitor and manage agents."
- `list_cmd` command (named "list"): `--pipeline-id` required option. Calls `client.list_agents(pipeline_id)`, formats with `format_agent_table`.
- `logs` command: `pipeline_id` argument. Calls `asyncio.run(stream_logs(base_url, pipeline_id, token, Console()))`. This blocks streaming until Ctrl+C.
- `restart` command: `agent_id` argument. Posts restart to API.

**5. Create `apps/cli/codebot/commands/config.py`** -- configuration:
- `config_group = click.group("config")` with docstring "Manage CLI configuration."
- Config file path: `~/.codebot/config.yaml`
- Helper `_load_config()` reads YAML or returns defaults `{"server_url": "http://localhost:8000", "preset": "full"}`. Helper `_save_config(cfg)` writes YAML.
- `preset` command: takes `preset` argument (choices: full, quick, review-only). Saves to config. Prints `"[green]Preset set to {preset}[/green]"`.
- `show` command: reads config, prints as Rich Panel.
- `set_cmd` command (named "set"): takes `key` and `value`. Valid keys: server_url, preset, token. Updates config.
