<objective>
Build the context compressor (multi-strategy compression) and the ContextAdapter (main entry point that orchestrates L0/L1/L2 assembly with budget enforcement).

Purpose: This is the integration layer -- the ContextAdapter is the single interface that every agent invocation uses to get its context. It wires together ThreeTierLoader (Plan 01), VectorStore + CodeIndexer (Plan 02), and the new ContextCompressor into a complete pipeline. Without this, agents would need to manually coordinate context assembly.

Output: compressor.py with priority-eviction and LLM-summarization strategies, adapter.py that assembles all context tiers, and comprehensive unit tests.
</objective>

       ```python
       from collections.abc import Callable, Awaitable
       SummarizerFn = Callable[[str], Awaitable[str]]
       ```
       This allows injecting any LLM summarization function without coupling to a specific LLM library. In tests, a mock that returns truncated text. In production, a function that calls litellm or anthropic to summarize.

       Class `ContextCompressor`:
       - `__init__(self, summarizer: SummarizerFn | None = None)`:
         - Stores summarizer callable. When None, compression only does priority eviction.

       - `async def compress(self, context: AgentContext) -> CompressionResult`:
         1. Record `original_tokens = context.total_tokens`
         2. **Stage 1 -- Evict LOW priority items:**
            Call `context.remove_items_by_priority(Priority.LOW)` to reclaim tokens.
            Track count of dropped items.
         3. Check `context.is_over_budget()`. If within budget, return CompressionResult.
         4. **Stage 2 -- Summarize MEDIUM priority items (if summarizer available):**
            For each item in context.items where priority == MEDIUM:
            - Call `summarized = await self.summarizer(item.content)`
            - Replace item content with summarized text
            - Recalculate token count for the item
            Track count of summarized items.
            If no summarizer, skip this stage.
         5. Check `context.is_over_budget()`. If within budget, return CompressionResult.
         6. **Stage 3 -- Summarize HIGH priority items (if summarizer available):**
            Same as stage 2 but for HIGH priority items.
         7. Check `context.is_over_budget()`. If still over after all stages, log a warning but do NOT drop CRITICAL items. Return CompressionResult with final token count.
         8. Return `CompressionResult(original_tokens=original_tokens, compressed_tokens=context.total_tokens, items_dropped=dropped_count, items_summarized=summarized_count, dropped_sources=dropped_ids)`

    1. Create `apps/server/src/codebot/context/adapter.py`:

       Class `ContextAdapter`:
       - `__init__(self, *, agent_role: str, token_budget: int, loader: ThreeTierLoader, compressor: ContextCompressor, vector_store: VectorStoreBackend | None = None, code_indexer: CodeIndexer | None = None, model: str = "gpt-4o")`:
         Stores all dependencies. vector_store and code_indexer are optional for graceful degradation.

       - `async def build_context(self, task: TaskSchema, agent_system_prompt: str = "", pipeline_phase: str = "") -> AgentContext`:
         ```
         context = AgentContext(budget=self.token_budget, model=self.model)

         # Step 1: L0 -- CRITICAL priority, always loaded
         l0 = await self.loader.load_l0(
             agent_system_prompt=agent_system_prompt,
             pipeline_phase=pipeline_phase,
         )
         l0_text = self._format_l0(l0)
         context.add(l0_text, priority=Priority.CRITICAL, source="l0")

         # Step 2: Task data -- HIGH priority
         task_text = self._format_task(task)
         context.add(task_text, priority=Priority.HIGH, source="task")

         # Step 3: L1 -- MEDIUM priority, loaded per phase/role
         l1 = await self.loader.load_l1(
             phase=pipeline_phase,
             agent_role=self.agent_role,
         )
         if l1.phase_requirements:
             context.add(l1.phase_requirements, priority=Priority.MEDIUM, source="l1_requirements")
         if l1.architecture_decisions:
             context.add(l1.architecture_decisions, priority=Priority.MEDIUM, source="l1_architecture")

         # Step 4: L2 -- LOW priority, vector store retrieval (optional)
         if self.vector_store is not None and context.has_budget(reserve=500):
             try:
                 # Generate query embedding (placeholder: use zero vector)
                 # In production, this would use sentence-transformers
                 query_embedding = [0.0] * 384
                 results = await self.vector_store.query(
                     query_embedding=query_embedding,
                     top_k=5,
                     filter={"project_id": str(task.project_id)},
                 )
                 for result in results:
                     if context.has_budget():
                         context.add(result.content, priority=Priority.LOW, source="l2_vector")
             except Exception:
                 pass  # L2 is best-effort; failures don't break context assembly

         # Step 5: Compress if over budget
         if context.is_over_budget():
             await self.compressor.compress(context)

         return context
         ```
