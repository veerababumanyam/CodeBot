           f. For each non-conditional edge (source not a SWITCH node): `builder.add_edge(edge.source, edge.target)`

           j. `return builder.compile(checkpointer=checkpointer)`

         - `_build_node_function(self, node_def: NodeDefinition) -> Callable`:
           a. For SWITCH nodes: return an async function that just passes through state (routing is handled by conditional_edges): `async def switch_fn(state: dict) -> dict: return {"node_outputs": {node_def.id: {"type": "switch", "evaluated": True}}}`
           b. For TRANSFORM nodes: return async function that reads `node_def.config.get("transform_fn")` key name and applies identity if not found
           c. For MERGE nodes: return async function that collects outputs from upstream nodes into a combined dict
           d. For GATE nodes: return an async function that reads `node_def.config.get("conditions", [])` -- a list of dicts each with `"key"` (dot-path into state, e.g. `"node_outputs.analyzer.status"`), `"operator"` (one of `"eq"`, `"neq"`, `"in"`, `"exists"`), and `"value"` (expected value). The function evaluates each condition against the current state:
              - `"exists"`: check that the key path resolves to a non-None value in state
              - `"eq"`: check that the resolved value equals `condition["value"]`
              - `"neq"`: check that the resolved value does not equal `condition["value"]`
              - `"in"`: check that the resolved value is in `condition["value"]` (a list)
              If ANY condition fails, raise `GateFailedError(f"Gate '{node_def.id}' failed: condition on '{condition['key']}' not met (expected {condition['operator']} {condition.get('value', '')}, got {actual_value})")`.
              If all conditions pass, return `{"node_outputs": {node_def.id: {"type": "gate", "status": "passed", "conditions_checked": len(conditions)}}}`.
           e. For all other node types (AGENT, SUBGRAPH, etc.): return a **stub** async function that records node_id in node_outputs: `async def stub_fn(state: dict) -> dict: return {"node_outputs": {node_def.id: {"status": "executed", "type": node_def.type.value}}}`. These stubs will be replaced by real agent/subgraph implementations in Phase 3.

         - `_build_switch_router(self, switch_node: NodeDefinition, edges: list[EdgeDefinition]) -> Callable`:
           a. Extract `cases` dict from `switch_node.config["cases"]` (maps case_name -> condition_expression)
           b. Extract `default` from `switch_node.config.get("default")`
           c. Build a mapping from case_name to target_node_id by matching edge conditions
           d. Return a function `router(state: dict) -> str` that:
              - Reads `state["node_outputs"].get(switch_node.id, {})` for any routing hints
              - Evaluates each case against state using a simple key-path comparator (e.g., `state.get("complexity") == "low"`)
              - For this phase, use a registry-based approach: condition strings like `"case == 'simple'"` are matched against the case names. The router checks `state["node_outputs"][switch_node.id]` for a `"route"` key, or falls back to evaluating `state.get(key) == value` for conditions of the form `"state.key == 'value'"`
              - Returns the target node_id for the matched case, or the default target
